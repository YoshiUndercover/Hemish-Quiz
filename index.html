1. Networking Basics

Types of LANs

  • SOHO (Small Office/Home Office): Simple design, limited hosts, uses standard WPA, standalone access points, operates flat networks (no subnets) with unmanaged switches. Uses low-power IoT protocols like Z-Wave.
  • Enterprise LANs: Hierarchical architecture, lightweight APs (LWAP) managed by a centralized WLC (Wireless LAN Controller), uses VLANs (Layer 2 segmentation via 802.1Q tags), 802.1X/RADIUS authentication.
  • Datacenter LANs: Ultra high-speed (40/100+ Gbps), redundant links, uses Spine-Leaf structures, blade server consolidation, SAN and NAS storage.

VLANs vs VXLANs

Feature VLAN (802.1Q) VXLAN (Virtual Extensible LAN)
Network ID Size 12-bit ID 24-bit VNI (VXLAN Network Identifier)
Max Networks 4,096 logical networks Over 16 million logical networks
Routing Method Trunk ports (802.1Q tags) across switches MAC-in-IP encapsulation (Layer 3 overlay tunnels via VTEP)
Primary Use Standard enterprise networks Large scale datacenters and cloud providers

VLAN Security Attacks

  • Switch Spoofing: Ripped switches establish trunking links with other switches by sending mock Dynamic Trunking Protocol (DTP) negotiations, granting access to all VLANs.
  • Double Tagging: Attacker prepends two VLAN tags. The first switch removes the outer tag and immediately forwards the frame to the next switch, which drops it into the target VLAN (inner tag) without check.

3. Troubleshooting & CLI Tools

6-Step CompTIA Methodology

  1. Identify the problem: Gather symptoms (S), identify affected areas/duplicate, determine if anything Changed, recognize Consequential effects (SICC).
  2. Establish a theory of probable cause: Evaluate obvious causes, research standard databases.
  3. Test the theory to determine cause: Validate the theory. If failed, establish a new theory.
  4. Establish a plan of action: Outline resolution tasks, identify potential side effects, and implement the solution.
  5. Verify full system functionality: Check complete system, implement preventive measures.
  6. Document findings: Note actions taken, lessons learned, and outcomes in a log database.

CLI Diagnostic Commands

  • ping: Sends ICMP Type 8 (Echo Request) and waits for ICMP Type 0 (Echo Reply) to verify reachability.
  • tracert (Windows) / traceroute (Unix): Maps the route to a host by sending packets with incrementing TTL values, recording the source IP of returning ICMP Type 11 (Time Exceeded) warnings.
  • ipconfig (Windows) / ifconfig (Unix): Shows IP settings, subnet masks, DNS servers, and gateways for active network adapters.
  • netstat: Lists active TCP/UDP socket connections, listening ports, and ethernet traffic logs.
  • nslookup: Queries name servers directly to resolve domain names to IP addresses (or reverse PTR lookups).

4. Network Topologies & Cloud Architecture

Topologies Reference

  • Star: Central switch/hub connected to all devices. Most common and reliable for SMBs. Single point of failure is the central switch.
  • Mesh: Every node connected (Full) or partially connected (Partial). Max redundancy, very expensive. Formula: n(n-1)/2 links.
  • Bus: Single backbone cable with terminators. Collisions are high (uses CSMA/CD). Obsolete.
  • Leaf-Spine: Two-tier architecture for datacenters. Every leaf (aggregators connected to hosts) links to every spine (high-speed core switch backbone).

Cloud Delivery Models

  • IaaS (Infrastructure as a Service): Provider handles hardware, virtualization, and storage. User manages OS, applications, and network firewalls (e.g. AWS EC2, Azure VMs).
  • PaaS (Platform as a Service): Provider manages OS, virtualization, and hardware. User gets deployment suites and APIs to run code without managing servers (e.g. Google App Engine).
  • SaaS (Software as a Service): Provider handles the entire stack. User interacts with prebuilt software directly over a browser (e.g. Google Workspace, Office 365).

5. Network Security, Cryptography & AAA

AAA Framework

  • Authentication: Proves identity (credentials, certificates, tokens).
  • Authorization: Dictates what actions or access levels the identity possesses (via ACLs, RBAC, ABAC).
  • Accounting: Tracks and logs what resources the identity consumed (auditing logins, commands).

Access Control Models

  • RBAC (Role-Based): Access assigned to standard business roles (e.g. Administrator, Manager).
  • ABAC (Attribute-Based): Context-aware permissions checking dynamic attributes (e.g. login time, host location, device posture).
  • DAC (Discretionary): The resource owner (creator of a file) determines who gets read/write access.
  • MAC (Mandatory): System-enforced access based on classification labels (e.g. Secret, Top Secret) and clearable levels. Admin-controlled.

Port Security Violation Actions

  • Protect: Drops offending packets. Operates silently without notifications or incrementing counters.
  • Restrict: Drops offending packets, increments the security violation count, generates a Syslog message/SNMP trap.
  • Shutdown: Instantly transitions the port to the err-disable state, turning it off. Requires administrator intervention to reset.

PoE (Power over Ethernet) Specs

  • IEEE 802.3af (PoE): Delivers up to 15.4W of DC power.
  • IEEE 802.3at (PoE+): Delivers up to 30W of DC power. Ideal for PTZ cameras and basic APs.
  • IEEE 802.3bt (PoE++ / Type 4): Delivers up to 60W or 90W of DC power. Supplies high-power smart devices.

8. Dynamic Routing Protocols

OSPF (Open Shortest Path First)

  • Class: Link-State routing protocol. Dijkstra algorithm constructs shortest path.
  • OSPF LSAs (Link State Advertisements):
    • Type 1 (Router LSA): Describes local router interfaces and link states within an area.
    • Type 2 (Network LSA): Sent by Designated Routers (DR) detailing nodes on a shared segment.
    • Type 3 (Summary LSA): Sent by Area Border Routers (ABR) to advertise routes between different areas.
    • Type 5 (AS External LSA): Advertises routes imported from external systems (by ASBR).
  • Areas: Hierarchical structure. Area 0 is the backbone. All other areas must connect directly to Area 0 via an ABR.

EIGRP & BGP

  • EIGRP: Advanced distance-vector (hybrid). Uses the DUAL algorithm. Metrics are bandwidth and cumulative delay. Fast convergence via precalculated backups called Feasible Successors. Supports unequal cost load balancing.
  • BGP: Path-vector protocol. BGP acts as the 'Protocol of the Internet', exchanging routes between Autonomous Systems (AS) over TCP port 179. Highly configurable policies.

9. AppleTalk Protocol Suite

AppleTalk is a legacy proprietary suite of protocols developed by Apple for networking Macintosh systems, structured into layers mapping directly to the OSI model.

Data Link & Network Layers (Layers 2 & 3)

  • LLAP (LocalTalk Link Access Protocol): Layer 2 protocol handling physical frames over original serial cabling.
  • EtherTalk/TokenTalk/FDDITalk: Adapters allowing AppleTalk to transport frames over standard network cabling.
  • AARP (AppleTalk Address Resolution): Resolves logical network nodes to physical hardware MAC addresses.
  • DDP (Datagram Delivery Protocol): Core best-effort socket routing protocol (Layer 3).

Transport & Session Layers (Layers 4 & 5)

  • RTMP (Routing Table Maintenance): Periodic distance-vector routing table broadcasts.
  • AURP (Update-Based Routing): Sends routing announcements only when a change occurs, replacing periodic broadcasts.
  • NBP (Name Binding Protocol): Maps human-readable device tags to logical numbers (DNS function).
  • ATP (Transaction Protocol): Standard reliable request-response transaction execution.
  • AEP (Echo Protocol): Tests connectivity and round-trip times (Ping function).
  • ZIP (Zone Information Protocol): Collects network numbers into logical zone groupings.
  • PAP (Printer Access Protocol): Manages connections to network printer queues.
  • ADSP (Data Stream Protocol): Full-duplex connection-oriented stream controller.

10. Backups, UPS & Server Infrastructure

Backup Types

  • Full Backup: Copies all files, regardless of change status. Resets the archive bit. Slowest to write, fastest to restore (1 file).
  • Incremental Backup: Copies only files changed since the last backup of any type. Resets the archive bit. Fastest to write, slowest to restore (requires last full and every incremental in sequence).
  • Differential Backup: Copies files changed since the last full backup. Does not reset the archive bit. Medium write speed, fast restore (requires last full and the final differential file).

UPS (Uninterruptible Power Supply) Types

  • Standby (Off-line): Simplest and cheapest. Devices run off utility power. Inverter kicks in when power drops. Switch time: ~10ms. Best for simple workstations.
  • Line-Interactive: Contains a variable transformer that regulates voltage sags and spikes without switching to battery. Switch time: 2–6ms. Best for basic server rooms.
  • Online Double Conversion: Utility AC is constantly converted to DC to charge batteries, then back to AC for clean output. Zero switch time (0ms). Maximum security, highest cost. Required for critical datacenters.
